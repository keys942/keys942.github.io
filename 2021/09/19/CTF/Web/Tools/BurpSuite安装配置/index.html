BurpSuite安装配置

BurpSuite配置激活

1、下载、解压完成后双击打开 burp-loader-keygen.jar 。

工具运行需要 Java 环境，请自行安装，此处不赘述。

2、修改 License Text 为任意值，这里我改为 wzz ，点击 run 启动 burp suite 。

如果上述方式没有成功启动 burpsuite，打开 cmd，进入 burp-loader-keygen.jar 所在目录，执行 Loader Command ：java -Xbootclasspath/p:burp-loader-keygen.jar -jar burpsuite_pro_v1.7.33.jar 。

成功打开 burpsuite 。

3、将 Keygen 中 License 复制粘贴到 burpsuite 中，点击 Next ，然后点击 Manual activation 。

4、点击 Copy request ，将字符串粘贴到 keygen 的 Activation Request 里，然后 Activation Response 框里就有了结果，再复制 Activation Response 中的内容，将字符串粘贴到 BurpSuite 剩下的那个框里，再点击 Next 显示激活成功，点击 finish 完成 。

5、然后点击 Next ，再点击 Start Burp ，得到burpsuite 主界面 。

6、激活之后，不能通过双击 BurpSuite 主程序来启动，否则启动之后还是需要重新激活，两个文件必须放在一个目录下，启动方式有两种：

  • 通过 keygen 上的 run 按钮来启动

  • 在文件目录下执行 java -Xbootclasspath/p:burp-loader-keygen.jar -jar burpsuite_pro_v1.7.33.jar 来启动 。为了方便，可以将命令保存为一个 bat 。

BurpSuite设置代理

这里推荐火狐浏览器来配合 burp 使用 。

1、设置浏览器代理

1）打开设置，搜索代理，点击设置 。

2）改为手动配置代理 ，设置如图所示 。

3）输入 127.0.0.1:8080 ，点击回车，进入 burp 证书下载页面，点击右上角，下载证书 。

4）打开设置，搜索证书，点击查看证书

5）点击导入，导入刚刚下载的 burp 证书，确定，这样就可以使用 burp 抓取 https 请求了 。

代理设置成功之后，访问网页是访问不了的，因为我们没有将请求转发，接下来就需要配置 burp 来转发浏览器的请求。
