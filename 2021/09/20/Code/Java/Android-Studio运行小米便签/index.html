Android Studio运行小米便签

Android Studio运行小米便签

1、源码下载

从 GitHub 或 Gitee 上下载源码压缩包后，进行解压，得到小米便签的源码 。

GitHub：MiCode/Notes: 小米便签社区开源版（Community edition of XM notepad/MIUI notes) (github.com)

Gitee：Notes: MiCode 便签是小米便签的社区开源版，由MIUI团队(www.miui.com) 发起并贡献第一批代码，遵循NOTICE文件所描述的开源协议 这个应用从Android开发者角度 (gitee.com)

2、导入项目

小米便签的源代码最后修改时间是在12年，那时候的代码是使用 Eclipse 开发的，因此，需要将其转换成能在 Android studio 下运行的 gradle 项目 。

1）在 Android studio 上，依次点击 File -> New -> Import Project ，选择刚才解压的小米便签的源码 Notes-master ，OK 。

2）选择导入到的目录，这里我选择默认，Next ，可能会提醒目录不存在是否创建，选择 Create 创建就好 。

3）去掉上面两个选项的勾勾 ，Finish ，等待加载完成。

3、配置构建Gradle

1）第一次构建可能会出现如下问题：

对如图选中的 build.gradle 进行修改 。

这是我的配置，添加了 google() ，如果添加了还是不可以或者想快一些，可以试试换成阿里的镜像 。（将 jcenter() 和 google() 换成我注释的那四句 ）

2）配置完成后，重新构建，出现两个警告，点进去查看 。

第一个显示 minsdk 不应该被声明 ，点进去，把它注释掉 。

第二个显示要用 Java 11，File -> Settings -> Build,Execution,Deployment -> Build Tools -> Gradle ，在 Gradle JDK 中选择 Android Studio Default JDK 。

3）成功构建 。

4、运行

1）点击小锤子，创建项目，出现错误。

这个错误应该是版本问题导致的，毕竟小米便签好多年前的东西了，对于这个问题的解决不唯一吧，可以下载相关组件，替换过时不再用的函数，也可以对版本进行回退。由于我比较懒，加上刚开始学习 Java，这里我选择版本回退 。

2）版本回退

具体的版本要求可以去官方查看，这里只是简单的列举了Gradle 版本和Gradle 插件版本之间的对应关系，实际应用还是要看 AS 版本支持的版本的，所以最好去官网看一下，要不然像我一样瞎试的话，可能会挤爆C盘 。

官网：Android Gradle 插件版本说明 | Android 开发者 | Android Developers (google.cn)

在相应的配置文件里进行修改 ，修改完后点击 Sync Now 进行同步。

3）重新构建之后有警告，显示 Build Tools 版本不行，尝试换一个版本，改成 29.0.2 ，改了之后还是有些警告，尝试将 compileSDKVersion 改为 22 ，点击 Sync Now 进行同步，就没问题了。

compileSDKVersion 改为 22 后，可能会报错，按照提示安装相应的包就好了 。

4）最后进行运行，创建项目没问题，但运行时，虚拟机启动了但一直显示 Waiting for all target devices to come online 同时APP界面没有出现，我猜测是内核不是很支持，所以我从一开始用的 29 内核 换了一个 22 内核，结果就出来了 。当然问题可能不是这个导致的，这里就不去深究了 。
