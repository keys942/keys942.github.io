CTFHub-Web-Web前置技能-HTTP协议

1、请求方式

可以看到，它告诉我们 HTTP Method是 GET，使用 CTF**B Method，它就会给我们 flag，这里没有直接给出 Method ，但可以猜测大概率会是 CTFHUB ，我们将 Method 进行修改，看能否得到 flag。

注意，它给了我们一个 Hint（提示）：如果得到 HTTP Method Not Allowed 错误，我们需要请求 index.php。所以，我们可以直接请求 index.php 来避免出现错误。

1）Curl

得到 flag：ctfhub{8edef26a2f40c879b279f243} 。

2）Burp

抓包

将这个包发到重放模块（Repeater）：修改 Method
发送后就可以得到flag

或

更改请求方式，更改完成后点击 Forward，放包
查看网页返回情况，成功拿到 flag

2、302 跳转

点击 give me flag 发现又回到了这个页面。

查看网页源码可知，点击 give me flag 应该跳转到 index.php ，实际上却跳转到了 index.html ，看来是被重定向了。

所以，我们可以使用 curl 命令行来查看响应，也可以使用 burp 抓包来改成 index.php 。

1）Curl

得到 flag：ctfhub{bca0c758b9ce7a4db57631d6}。

2）Burp

使用 burp 抓包，将这个包发到重放模块；

将 index.html 修改为 index.php ;

发送，即可得到 flag 。

3、Cookie

根据题目，我们必须要伪造成管理员才能得到 flag，可以通过改 cookie 来伪造成管理员。

1）Burp

使用 burp 抓包，可以看到 cookie: admin = 0

将这个包发到重放模块，将其修改为 cookie: admin = 1 。

发送，得到 flag 。

2）HackBar

查看得知 Cookie：admin = 0

点击 Load URL ，载入 URL

勾选 Cookies ，输入 admin = 1

点击 Execute ，运行，得到 flag：ctfhub{3dd0714df3ce639635c048d7} 。

4、基础认证

进入题目页面，发现点击 click 会出现认证登录页面，先任意输入 admin/admin 进行登录尝试，发现不对。抓包看数据，可以看见一条特殊字符串 WWW-Authenticate：Basic realm = "Do u know admin ?" ，暂时没有其他线索，假设用户名就是 admin，然后进行暴力破解。

Basic 为验证的模式，表示基础认证，realm = “Do u know admin ?” 为保护域

再次输入 admin/admin 进行登录尝试，对认证请求进行抓取，得到认证消息 Authorization：Basic YWRtaW46YWRtaW4= ， YWRtaW46YWRtaW4= 大概率为 Base64 编码 。将 YWRtaW46YWRtaW4= 发到 Decoder ，对其进行 Base64 解码，得到开始尝试输入的账号和密码 admin:admin 。

接下来，对其进行暴力破解：

1）将报文发送到 Intruder，点击 Positions，将YWRtaW46YWRtaW4= 添加为 Payload Positions 。

2）在 Payloads 选项卡下，选择 Payload Type 为 Simple List，然后在 Payload Options 中点击 Load 加载密码字典。

3）在 Payload Options 中，点击 Add ，选择 Add Prefix（添加前缀） ，输入 admin: 。

​ 在 Payload Options 中，点击 Add ，选择 Encode（添加一个编码方式） ，选择 Base64 Encode 。

注意：一定要先加前缀 admin: 然后再进行编码，因为服务端判断的是合并后的 base64 编码。

4）在 Payload Encode 中，取消勾选的 URL-encode ，不然你会看到 base64 之后的 = 会被转换成 %3d ，你就算爆破到天荒地老也不会出来 。

5）点击 Start Attack ，开始爆破，完成后按 Length 排序，发现一个长度不一样的，并看到状态码为 200 ，说明爆破成功 。

6）点击查看，点击 Response，得到 flag：ctfhub{a6bb3b3c9d6d4eeab2dae1f3}

5、响应包源代码

根据题目，F12 查看源代码，得到 flag：ctfhub{15bf236024e2d2979ade7339}

当然也可以用 curl 和 burp 得到 flag 。
