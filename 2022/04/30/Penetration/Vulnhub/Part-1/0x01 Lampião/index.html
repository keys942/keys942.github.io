进行编译、运行：

  1. -Wall 一般使用该选项，允许发出GCC能够提供的所有有用的警告
  2. -pedantic 允许发出ANSI/ISO C标准所列出的所有警告
  3. -O2编译器的优化选项的4个级别，-O0表示没有优化,-O1为缺省值，-O3优化级别最高
  4. -std=c++11就是用按C++2011标准来编译的
  5. -pthread 在Linux中要用到多线程时，需要链接pthread库
  6. -o dcow gcc生成的目标文件,名字为dcow

获取到 root 的密码，通过 ssh 登录即可。

四、总结

1、cewl、hydra、msf 使用

2、信息收集：nmap 扫描对方端口和服务，dirb 爆破网站目录

3、权限获取：网站cms漏洞，得到 www-data shell；用户爆破，得到一般用户

4、提权：les.sh 扫描内核漏洞，通过 dirtycow 内核漏洞提权，两机传输文件，编译、运行，得到 root shell

5、完整的渗透流程：外网 -> 内网（网站漏洞 -> www-data shell -> 内核扫描 -> 内核漏洞 -> 内核提权）

6、网站扫描：2类工具（扫描类AWVS、爬虫类xray（ddosi））
